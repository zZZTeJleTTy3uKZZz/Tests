# База знаний технической документации API

Это Backend-система для централизованного учета, управления и взаимодействия со спарсенными техническими документациями.
Включает CRUD API и интеграции с RAG системами (NotebookLM, LightRAG).

## Интеграция с NotebookLM (HTTP REST API)

Платформа автоматически интегрируется с Google NotebookLM. Когда документ в БД переходит в статус `Загружается в RAG`, FastAPI отправляет HTTP-запросы к локальному сервису `notebooklm-mcp-server`, который:
1. Создает новый пустой блокнот в NotebookLM.
2. Проверяет размер исходного Markdown-файла. Если он превышает лимиты Google (190МБ или 490 000 слов), файл автоматически разбивается на логические части.
3. Загружает `.md` файл(ы) документации как источник(и).
4. Возвращает ID блокнота для сохранения в БД.

> ⚠️ **ВАЖНО**: Для работы интеграции вы должны **один раз авторизоваться** в своем Google-аккаунте через инструмент CLI скилла NotebookLM или через эндпоинт `/setup-auth` на порту 3030.

## Инструкция по запуску с помощью Docker

Убедитесь, что у вас установлен `docker` и `docker-compose`.

```bash
cd "База знаний технической документации/src"
docker-compose up -d --build
```

**Доступные сервисы:**
- **FastAPI Бэкенд**: http://localhost:8000 (Документация Swagger: http://localhost:8000/docs)
- **NotebookLM HTTP API**: http://localhost:3030

## Инструкция по локальному запуску (без Docker)

Вам потребуется установленный Python 3.11+ и пакетный менеджер `uv`.

```bash
cd src/
uv sync
uv run alembic upgrade head
uv run uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```
